基本

Q. 非エンジニアでも参加できますか？

A. はい。初心者歓迎です。まずはキックオフで全体像をつかみ、何を任せたいか を1つ決めるところから始めれば大丈夫です。

Q. 使うツールに制限はありますか？

A. 基本自由です。Claude Code, GPT, Gemini, Cursor, OpenClaw など、使いたいものを自由に組み合わせてOKです。

Q. 何を作ればいいですか？

A. 自律的に動くAIエージェント がテーマです。一度お願いしたら最後まで進められる、途中で立て直せる、足りない情報を自分で集めにいける、といった性質があると今回のテーマに合いやすいです。

Q. 今の時点で完成形まで決めていないとダメですか？

A. ダメではありません。キックオフを終えた時点で、試してみたい方向性 と 最初の一歩 が見えていればOKです。

Q. キックオフを見逃しても参加できますか？

A. はい。録画アーカイブと参加者ガイドを見て、👋-自己紹介・意気込み と 📋-times から始めてもらえればOKです。

Q. どこで質問すればいいですか？

A. ❓-質問・相談 で聞いてください。進捗共有は 📋-times、もくもく作業は 🔇-もくもく作業 を使う想定です。

提出

Q. 提出方法は何ですか？

A. 現時点では次の2つを提出物の基本と考えています。

  • 作品の Skills / コード / README を入れた GitHub公開 repo
  • 3分以内のデモ動画

Q. GitHub repo は公開にしないとダメですか？

A. はい。最終提出時点では、作品の中身が分かる GitHub公開 repo での提出をお願いしています。

Q. デモ動画はどこに置けばいいですか？

A. なんでもOK です。共有しやすく、審査側が見られる形であれば大丈夫です。

Q. どんな形式の repo にすればいいですか？

A. 作品の中身が分かる GitHub公開 repo で大丈夫です。そのうえで、Claude Code で他の人がすぐ試せる形にしたい場合は、Plugin Marketplace形式 がおすすめです。

最低限、README、実装または SKILL.md、試し方 が入っていると伝わりやすいです。

Q. GitHubに慣れていなくても大丈夫ですか？

A. 大丈夫です。作品の Skills / コード / README が入った GitHub 公開 repo としてまとめてもOKですし、テンプレ repo を fork して進めてもOKです。

Q. Demo Day では全員が発表しますか？

A. すべての提出作品を見たうえで、ファイナリスト5チーム が Demo Day で発表する想定です。

Q. 審査員は誰ですか？

A. 審査員は現在最終調整中です。確定次第、Discordでご案内します。

Q. 審査基準は何ですか？

  • 自律性 40%
  • クオリティ 35%
  • インパクト 25%

チーム参加

Q. チーム参加はできますか？

A. はい。個人でもチームでも進めて大丈夫です。作品として分かりやすい形でまとまっていれば問題ありません。途中でチーム化したりメンバー変更があった場合は、運営に一言共有してください。

Q. チーム提出はどうすればいいですか？

A. 代表者1名 の提出でOKです。

Q. 1チームで複数のSkillを提出してもいいですか？

A. 提出は 1チームにつき1作品まで でお願いします。そのため、チームとして別々の複数作品を出す形は不可です。

一方で、1つの作品としてまとまっている のであれば、その中に複数のSkillを含める形は問題ありません。最終的に、1つのGitHub公開 repo と 1本のデモ動画 にまとまっていることを目安にしてください。

Q. チーム内でそれぞれ別のSkillを担当しても大丈夫ですか？

A. はい。チーム内で分担して進めること自体は問題ありません。ただし、最終提出は 1つの作品 として伝わる形にまとめてください。
